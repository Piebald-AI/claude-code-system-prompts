ID 1: Agent Prompt: Plan Mode (Enhanced) (agent-prompt-plan-mode-enhanced.md)
Category: Agent Prompts | Status: Adopted & Highly Adapted
Description: System instructions for the Plan subagent, guiding architectural design and codebase analysis.
Audit Details: In Claude Code, this prompt instructs a specialized 'Plan' subagent to explore file structures, evaluate architectural styles, and propose detailed designs. It emphasizes tracing dependencies and compiling a step-by-step strategy. It is designed to run in a separate, isolated subagent container.
Gemini Rationale:
In Antigravity, we utilize Gemini Pro's massive 2M token context window, eliminating the need to isolate the planner in a separate docker container to save context. However, Gemini models are susceptible to instruction dilution when loaded with too much conversational prose or redundant warnings. This requires converting the detailed architectural principles into explicit directives and strict tool-usage policies.
Gemini Strategy:
We integrate these planning rules into the Antigravity `planning-and-verification` skill. To enforce the read-only boundary during exploration, the agent's write tools (`write_to_file`, `replace_file_content`, `multi_replace_file_content`) are disabled, and a `pre_tool_call_decide` hook blocks state-altering shell commands (like `touch`, `mkdir`, `git add`, `npm install`). We also register a post-turn validation hook that rejects the turn unless the response ends with the structured '### Critical Files for Implementation' section listing exactly 3-5 files.

ID 2: System Reminder: Plan Mode is Active (5-Phase) (system-reminder-plan-mode-is-active-5-phase.md)
Category: System Reminders | Status: Adopted & Enforced
Description: System reminder injected to keep the agent inside the strict 5-phase gated planning pipeline.
Audit Details: Instructs the coordinator model to systematically proceed through five distinct phases: 1. Exploration (reading codebase), 2. Design (proposing alternatives), 3. Review (auditing solutions), 4. Plan compilation (writing implementation_plan.md), and 5. Approval (calling ExitPlanMode).
Gemini Rationale:
Without a structured gating pipeline, Gemini tends to skip exploration and immediately start editing source files, resulting in broken builds and redundant code. Enforcing a multi-phase workflow ensures that existing helpers and codebase conventions are identified before modification.
Gemini Strategy:
We implement the 5-phase planning flow directly in the Antigravity `planning-and-verification` skill, managed by the coordinator state machine. Rather than spawning expensive external subagents for exploration (Phase 1) and design (Phase 2), we leverage Gemini Flash's parallel tool calling to run up to 10 codebase searches (`grep_search`, `list_dir`) simultaneously. We restrict writing permissions exclusively to `implementation_plan.md` throughout the planning phase.

Category: System Reminders | Status: Adopted & Enforced
Description: Enforces that the planning turn must terminate exclusively with a specific approval tool call.
Audit Details: Strictly forbids the agent from asking if the plan is okay in raw text or using informal questions. The agent must call 'ExitPlanMode' to request plan approval or 'AskUserQuestion' to choose between specific designs. Any other termination triggers a system rejection.
